Project Overview

Note: This is an enterprise-grade authentication microservice built with Go, Gin, PostgreSQL, and Redis. It provides comprehensive security features including JWT authentication, 2FA, device tracking, and geolocation-based access control.

What is this?

A production-ready authentication microservice that handles all user authentication, authorization, and security operations. It implements industry best practices for password security, token management, and access control.

Core Technologies

  • Backend: Go 1.23+ with Gin Web Framework
  • Database: PostgreSQL 12+ for persistent storage
  • Cache: Redis 6.0+ for sessions and caching
  • Authentication: JWT (HS256) with refresh token pattern
  • 2FA: TOTP (RFC 6238) with QR code generation

Base URL

http://localhost:8080

Features

JWT Authentication

Two-Factor Authentication

TOTP support with Google Authenticator + 12 backup recovery codes.

Bcrypt+pepper hashing, complexity validation, breach checking, and history tracking.

Track devices, geolocation, browser info, and last login timestamp.

6-digit codes with 15-minute expiration for signup and password reset.

Security Features

1. Password Security

Storage: Bcrypt with cost factor 12 + pepper

Password Hash Format: $2a$12$[hash] Algorithm: bcrypt(password + pepper, 12)

Validation Requirements:

  • Length: 12-128 characters
  • 3+ uppercase letters
  • 3+ lowercase letters
  • 3+ numbers
  • 3+ special characters
  • Not in Have I Been Pwned database
  • zxcvbn strength score ≥ 2

History Tracking: Last 5 passwords stored, prevents reuse

2. JWT Token Security

Token Type: HS256 (HMAC-SHA256) signed

Access Token TTL: 15 minutes Refresh Token TTL: 7 days Token Version: Incremented on password change

Validation Checks:

  • Signature verification with secret key
  • Expiration time validation
  • Blocklist check in Redis
  • Token version matching
  • JWT ID (jti) uniqueness check

3. Two-Factor Authentication

  • TOTP: RFC 6238 with 30-second time window
  • QR Code: For authenticator app scanning
  • Backup Codes: 12 one-time recovery codes
  • Secret Storage: Encrypted in database

4. Rate Limiting & Brute Force Protection

Login Attempt Tracking:
1-2 attempts → Allow
3 attempts → CAPTCHA required
5+ attempts → 15-minute block

5. Geolocation Security

  • IP-based geolocation using ip-api.com
  • Country-level access restrictions
  • Location consistency validation
  • Impossible travel detection

6. Device Tracking

  • Unique device ID per login
  • Browser/User-Agent logging
  • Geographic coordinates (lat/lon)
  • Last login timestamp

7. Session Management

Redis Key: session:{userId}:{jti} TTL: 15 minutes (access token lifetime) Data: Device info, timestamps, flags

8. Constant-Time Comparison

Prevents timing attacks on:

  • Verification codes
  • Password hashing
  • Token validation

9. Email Security

  • 6-digit OTP with 15-minute expiration
  • Constant-time code comparison
  • Failed attempt tracking
  • SMTP over TLS

10. Reauthorization for Sensitive Ops

Before allowing password/email changes, user must reauthenticate:

  • Option 1: Enter current password
  • Option 2: Enter 2FA code
  • Valid for: 5 minutes
  • Sets Redis flag: "Reauth:{email}"

Database Schema

users Table

CREATE TABLE users ( id SERIAL PRIMARY KEY, created_at TIMESTAMP, updated_at TIMESTAMP, deleted_at TIMESTAMP, -- Authentication username VARCHAR(100) UNIQUE, email VARCHAR(320) UNIQUE, password_hash VARCHAR(255), token_version INT DEFAULT 0, -- Email Verification is_email_verified BOOLEAN DEFAULT FALSE, verification_code VARCHAR(255), verification_expires_at TIMESTAMP, -- 2FA tfa_code VARCHAR(255), tfa_verified BOOLEAN DEFAULT FALSE, login_codes TEXT, login_codes_set BOOLEAN DEFAULT FALSE, -- Profile name VARCHAR(255), phone VARCHAR(20) UNIQUE, gender VARCHAR(10), birthday DATE, role VARCHAR(20) DEFAULT 'user', is_active BOOLEAN DEFAULT TRUE, is_verified BOOLEAN DEFAULT FALSE, profile_image_url VARCHAR(500), cover_image_url TEXT, bio TEXT, provider VARCHAR(50) ); CREATE INDEX idx_users_email ON users(email); CREATE INDEX idx_users_username ON users(username);

device_record Table

CREATE TABLE device_record ( id SERIAL PRIMARY KEY, userid INT REFERENCES users(id), -- Location city VARCHAR(255), region VARCHAR(255), country VARCHAR(255), locale VARCHAR(10), lat FLOAT8, lon FLOAT8, zipcode VARCHAR(50), -- Device Info browser TEXT, last_login TIMESTAMP ); CREATE INDEX idx_device_record_userid ON device_record(userid);

old_password Table

CREATE TABLE old_password ( id SERIAL PRIMARY KEY, user_id INT REFERENCES users(id), password VARCHAR(255), created_at TIMESTAMP ); CREATE INDEX idx_old_password_user_id ON old_password(user_id);

Data Relationships

  • users (1) ← → (M) device_record - One user, many devices
  • users (1) ← → (M) old_password - One user, password history
  • Session (in Redis): Temporary session data with TTL

Authentication Flow

Signup Flow

  1. User POST to /user/signup with name, email, password, phone, gender, zipcode
  2. GeoGuard middleware validates geographic access
  3. Password validation (complexity, breach check)
  4. Generate 6-digit verification code
  5. Send code via email (Gmail SMTP)
  6. Store temporary data in Redis (15 min TTL)
  7. User receives verification code
  8. User POST to /user/verify_signup_email with code
  9. Verify code (constant-time comparison)
  10. Create user in database
  11. Hash password with bcrypt + pepper
  12. Add to password history
  13. Account created

Login Flow (with 2FA)

  1. User POST to /user/login with email and password
  2. Check rate limits (CAPTCHA after 3 attempts)
  3. Verify email format
  4. Find user by email
  5. Verify password with bcrypt
  6. Generate device record (geolocation)
  7. Generate 6-digit verification code
  8. Send verification code via email
  9. User POST to /user/verify-email with code
  10. Verify code
  11. Check if 2FA enabled
  12. If 2FA enabled:
    • User POST to /user/verify-2fA with authenticator code
    • Verify TOTP with current time window
  13. Generate JWT tokens (access + refresh)
  14. Create session in Redis
  15. Return tokens to user

Token Refresh Flow

  1. Access token expires (15 min)
  2. User POST to /user/refresh
  3. Retrieve refresh token from cookie
  4. Validate refresh token signature and expiration
  5. Check token version matches user version
  6. Generate new access token (15 min)
  7. Create new session
  8. Return new token

Password Change Flow

  1. User sends POST to /user/reauth-password with password
  2. Verify password
  3. Set Redis flag "Reauth:{email}" (5 min TTL)
  4. User sends POST to /user/change-password with new password
  5. Check Reauth flag exists
  6. Validate new password (complexity, history)
  7. Hash new password (bcrypt + pepper)
  8. Update database
  9. Add old password to history
  10. Increment token_version (invalidates all tokens)
  11. Delete all sessions (force re-login)

API Endpoints

Authentication Endpoints

POST /user/signup

Register new user account

Request: { "name": "John Doe", "email": "john@gmail.com", "password": "SecurePass123!@#", "phone": "1234567890", "gender": "male", "ZipCode": "94105" } Response: 202 { "message": "Verify your email to continue" }
POST /user/verify_signup_email

Confirm email and create account

Request: { "email": "john@gmail.com", "code": "123456" } Response: 200 { "user": { ... } }
POST /user/login

Authenticate with email and password

Request: { "email": "john@gmail.com", "password": "SecurePass123!@#" } Response: 200 { "message": "Enter verification code to continue" }
POST /user/verify-email

Complete login with email code

Response: 200 { "user": {...}, "token": "eyJhbGciOiJIUzI1NiIs...", "refresh_token": "eyJhbGciOiJIUzI1NiIs..." }

2FA Endpoints

POST /user/create-2fA

Setup two-factor authentication

Response: 200 { "email": "john@gmail.com", "png": "data:image/png;base64,iVBORw0KGgo...", "secret": "JBSWY3DPEBLW64TMMQ6AAAAA" }
POST /user/verify-2fA

Verify 2FA code

Installation & Setup

Prerequisites: Go 1.23+, PostgreSQL 12+, Redis 6.0+

Step 1: Clone Repository

git clone https://github.com/hossamelden202/GIN_AUTH.git cd GIN

Step 2: Install Dependencies

go mod download go mod tidy

Step 3: Setup PostgreSQL

# Create database psql -U postgres -c "CREATE DATABASE book;" # Create tables (from docs/schema.sql or manually) psql -U postgres -d book < docs/schema.sql

Step 4: Setup Environment Variables

# Create .env file HOST=localhost PORT=5432 user_name=postgres password=your_password db_name=book ssl=disable jwt_secret=your-long-secret-key-32-chars-min domain=localhost Mail_email=your-email@gmail.com Mail_password=your-app-password Pepper=your-pepper-24-chars-min opt_secret=your-otp-secret

Step 5: Start Redis

# macOS/Linux redis-server # Docker docker run -d -p 6379:6379 redis:latest

Step 6: Run Application

go run main.go # Output: # connected to database # [GIN-debug] Listening and serving HTTP on [::]:8080

Step 7: Access API Documentation

  • Swagger UI: http://localhost:8080/api-docs/
  • Health Check: http://localhost:8080/health
  • API Spec: http://localhost:8080/docs/swagger.yaml
